**Date**: 2026-04-22
**Status**: Accepted
**Supersedes**: none

## Context

Before writing any more implementation code for v0.1.0, we need a
stable public API. The v0.0.1 POC code (`sckit.go` prior to this ADR)
was written inside-out from the dylib â€” it exposed `Capture(displayID
uint32)` and `NewStream(displayID, w, h)` because those matched the C
calls. That shape does not scale to window/app/region targets.

## Decision

Adopt the API shape documented in `docs/API_DESIGN.md` verbatim:

- A sealed `Target` interface with concrete values: `Display`, `Window`,
  `App`, `Region`, `Exclude{Target, Windows}`.
- Every blocking call takes `context.Context` as first argument.
- Functional options (`WithResolution`, `WithFrameRate`, `WithCursor`,
  `WithColorSpace`, `WithQueueDepth`) instead of a config struct.
- `NextFrame(ctx) image.Image` for idiomatic callers; `NextFrameBGRA(ctx)
  Frame` for zero-copy hot paths.
- `Stream` implements `io.Closer`; close is idempotent; finalizer
  backstop logs a warning.
- Errors: sentinel `ErrTimeout`, `ErrPermissionDenied`,
  `ErrDisplayNotFound`, `ErrStreamClosed`; plus wrapped dylib errors.
- Convenience adapter `Stream.Frames(ctx) (<-chan image.Image, <-chan
  error)` built on top of `NextFrame`.

## Consequences

### Positive
- Target-based filter matches the underlying `SCContentFilter`
  abstraction 1:1 â€” new capture scopes (region, app, cursor-only) can
  be added by growing the `Target` family without touching existing
  callers.
- Options are additive â€” adding `WithColorSpace` in v0.3 doesn't
  break v0.1 users.
- Context.Context enables `ctx, cancel := context.WithTimeout(...)`
  out of the box. Agents need this.

### Negative
- Sealed interface means external code can't write its own Target
  (e.g. a mock for testing). We provide a `sckit.TestTarget` helper
  in tests to mitigate.
- Every public function gains a `ctx` param â€” churn for callers that
  don't care. We judge this acceptable because the "don't care"
  callers can pass `context.Background()` and forget about it.

### Required dylib changes
- New function `sckit_stream_cancel(handle)` to support ctx
  cancellation.
- New function `sckit_list_windows` + `sckit_capture_window` + stream
  variants for Window target.
- New function `sckit_list_apps` + composed-app capture for App target.
- All of these will be added in Days 3â€“4 per PLAN.md Â§6.

## Alternatives considered

### A1. Struct-with-many-fields config (rejected)
```go
sckit.Capture(sckit.Config{DisplayID: ..., Width: ..., ShowCursor: ...})
```
Pros: discoverable in editor autocomplete. Cons: API bumps require
recompile of every caller when new fields land.

### A2. Fluent builder (rejected)
```go
sckit.Display(id).Width(1920).ShowCursor(false).Capture(ctx)
```
Pros: reads nicely. Cons: requires stateful intermediate types; not
idiomatic in Go outside `text/template`-style APIs.

### A3. Bare integer display index (rejected)
```go
sckit.CaptureDisplay(0)
```
`kbinani` used this. Cons: display indices are not stable across
connect/disconnect/monitor-arrangement-changes. `CGDirectDisplayID`
(which is a `uint32` we expose via `Display.ID`) is stable.

### A4. No Context.Context (rejected)
Simpler for trivial use. But cancellation is a must-have for agents,
and retrofitting later is a breaking change. Do it right now.

## Soak period

API is declared frozen 2026-04-22. Any change before v0.1.0 tag
requires a new ADR (002+) referencing this one. Changes after v0.1.0
require a minor version bump and a deprecation path.
